
====== Инструкция по написанию статей ======

Данная статья написана для сотрудников **Escape Room Doctor** и содержит основные требования и правила оформления документа.

===== Имена файлов =====  

Необходимо правильно назвать файл. На первом этапе все статьи будет создавать администратор сайта и ставить задачи по написанию статьи, причем изначально все статьи пишем только на русском языке (в имени файла обязательно будет постфикс **\_rus**), например, эта статья имеет имя **howto\_guide\_rus**. При создании англоязычной версии статьи необходимо создать файл **С ТОЧНО ТАКИМ ЖЕ ИМЕНЕМ**, но с постфиксом **\_eng**.


===== Многоязычность =====  

В начале каждой статьи должен быть вот такой элемент:  
[{{:uplevel.png?direct&20|}}](contents_rus)
[{{:flag_rus.png?direct&20|}}](howto_guide_rus)
[{{:flag_eng.png?direct&20|}}](howto_guide_eng)

который позволяет выбрать язык статьи и переключиться на содержание. Делается этот элемент просто: берется вот такой шаблон  

``
[{{:uplevel.png?direct&20|}}](contetnts_rus)
[{{:flag_rus.png?direct&20|}}](FILENAME_rus)
[{{:flag_eng.png?direct&20|}}](FILENAME_eng)
``  

где FILENAME - это имя текущего файла.

===== Загрузка изображений =====  

Для загрузки нового изображения необходимо сначала создать эту картинку на локальном компьютере и дать ему понятное название, например:  arduino\_nano\_1.jpg, barrier\_sensor\_E18\_D80NK\_1.jpg, uno\_screw\_shield-1.jpg, servo\_fs5109m\_1.jpg и т.п., то есть название должно отражать суть картинки, но при этом быть относительно коротким, в имени должны быть только латинские буквы, цифры и символы **\_ и -**. Чтобы вставить картинку нужно нажать на значке "картинка" в меню {{:screen:dokuwikimenu.png?direct&400}}, выбрать папку, куда картинка будет сохранена: фотографии в папку **photo**, скриншоты - в папку **screen**, далее нажать "Select Files", выбрать локальный файл с компьютера и нажать **Upload**. 

**Для того, чтобы избежать возможной путаницы, примем, что в названиях файлов и ссылок пробелы будем заменять знаками "тире" -**
то есть newimg-dummy-1.png и тому подобное  

** ВАЖНО!!! Учитывая, что хранилище находится под контролем [Git](https://ru.wikipedia.org/wiki/Git), необходимо ограничивать размер файлов: для фотографий лимит около 100кб, для скриншотов - пару десятков кб максимум.**  

Если картинка ранее загружена в dokuwiki, то нужно просто выбрать эту картинку из списка загруженных и в появившемся окне нажать **insert**. Картинка вставится в таком формате:  
``
{{:screen:dokuwikimenu.png?direct&400}}
``

картинка вставлена, дплее при помощи параметра, следующим за знкчком &, можно поменять размер картинки. В нашем случае размер картинки равен 400.


===== Внутренние ссылки =====  

Для внутренних ссылок используется либо текстовая ссылка, например, **[ссылка](howto_guide_rus)**, код:  
``
**[ссылка](howto_guide_rus)**
``
либо ссылка с картинкой, например [{{:flag_rus.png?direct&20|}}](howto_guide_rus), код:  
``
[{{:flag_rus.png?direct&20|}}](howto_guide_rus)
``
в обоих случаях в круглых скобках указывается имя документа, на который ссылаемся, а саму ссылку нужно выделять **жирным** при помощи двух звездочек.  

Чтобы добавить свою собственную внутреннюю ссылку, нужно придумать для неё название и написать в коде, будто страница уже существует, например:
``
**[newpage_dummy]**(np_dummy)
``
и перейти по ссылке, что у нас указана в скобках (и красного цвета, что означает, что такой страницы не существует)
Далее переходим в режим редактирование ("Edit page") и пишем статью, после чего сохраняем. 
Готово, внутренняя ссылка создана и отображается в **[Sitemap](http://www.escaperoomdoc.com/doku.php?id=howto_guide_rus&do=index)**
Чтобы удалить внутреннюю ссылку, нужно удалить статью.
Чтобы удалить статью, нужно просто удалить весь текст из неё, после чего внутренняя ссылка автоматически пропадёт из **[Sitemap](http://www.escaperoomdoc.com/doku.php?id=howto_guide_rus&do=index)**.

===== Внешние ссылки =====  

Для создания внешних ссылок используется механизм, аналогичный текстовым внутренним ссылкам, например, **[visit google site](https://www.google.com)** только в место имени документа в круглых скобках указывается URL страницы, а саму ссылку нужно выделять **жирным** при помощи двух звездочек.

===== Цитата, код или  =====  

Для создания такой вот цитаты:

''
for ( int i = 0; i < 0x10; i ++ )   
{   
   obj[ i ].tryToPrepare( );   
   if ( obj[ i ].isReady( ) ) obj[ i ].process( );   
}   
''
используется символ ' (см.в режиме редактирования).
или
```xml
<response type='getstates' status='ok'>
  <item type='pwm' obj='pwm1' attr='state' value='128'/>
  <item type='out' obj='out1' attr='state' value='on'/>
  <...>
</response>
```
используется символ три символа ' и ключевое слово xml (см.в режиме редактирования).  


===== Специальные символы =====  

Markdown имеет набор специальных символов, которые нужно использовать строго для форматирования, например, нижнее подчеркивание, звездочка и их описание есть в стандарте, но если нужно отобразить этот символ, то перед ним необходимо поставить обратный слэш: \. Например \ \_, \ \* и т.д.
